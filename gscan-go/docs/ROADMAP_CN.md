# Go 重构路线图

## 当前状态

当前 Go 版已经具备原项目的主要扫描面，并且统一到了可测试、可扩展的工程结构里。

已完成：

- 模块化 CLI 与配置解析
- 扫描器接口与统一结果模型
- 规则库与 GeoIP 离线库复用
- Rootkit 规则运行时解析
- Webshell 规则运行时解析
- 核心扫描面迁移
- 文本报告、JSON 报告、差异哈希
- 定时任务写入
- 安全日志打包

## 下一阶段

目标：补齐原始项目能力缺口，提升兼容性与误报控制。

计划项：

- 改进 Linux 发行版兼容性
- 将 `ss` / `who` / `lastlog` 之类的系统依赖封装成更清晰的适配层
- 增加误报白名单机制
- 增强 Webshell 规则兼容性，逐步逼近完整 YARA 行为
- 增加更多集成测试样本和回归测试

## 长期方向

目标：从“扫描工具”升级为“主机应急平台能力底座”。

计划项：

- HTML 报告和时间线视图
- 规则热更新
- 差异分析增强
- API / agent 模式
- 插件式扩展扫描器

## 后续增强建议

优先级最高的下一步：

1. 增加 Linux 实机回归样本
2. 提升 Webshell 规则解释器兼容性
3. 引入 HTML 报告
4. 增加插件式扩展能力
