# Go 重构路线图

## Phase 1

目标：先拿到一版可运行、可验证、可继续增强的 Go CLI。

已完成：

- 模块化 CLI 与配置解析
- 扫描器接口与统一结果模型
- 规则库与 GeoIP 离线库复用
- 核心扫描面迁移
- 文本报告、JSON 报告、差异哈希

## Phase 2

目标：补齐原始项目能力缺口，提升兼容性与误报控制。

计划项：

- 迁移 Rootkit 全量特征数据
- 接入 Webshell YARA 扫描
- 改进 Linux 发行版兼容性
- 将 `ss` / `who` / `lastlog` 之类的系统依赖封装成更清晰的适配层
- 增加误报白名单机制

## Phase 3

目标：从“扫描工具”升级为“主机应急平台能力底座”。

计划项：

- HTML 报告和时间线视图
- 规则热更新
- 差异分析增强
- API / agent 模式
- 插件式扩展扫描器

## 后续增强建议

优先级最高的下一步：

1. 迁移 Rootkit 扫描
2. 迁移 Webshell 扫描
3. 增加集成测试样本
4. 优化 Linux-only 运行验证
